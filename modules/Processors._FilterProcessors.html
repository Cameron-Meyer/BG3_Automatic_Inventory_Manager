FilterProcessor:RegisterTargetStatProcessors (modUUID, statFunctions)
Adds the provided stat functions to the list of possible functions, using the key as the criteria, which process CompareFilters using mod-added If the targetStat identified already has a processor associated, ignore the provided one and continue TargetStats

Parameters:

  • modUUID that ScriptExtender has registered for your mod, for tracking purposes - https://github.com/Norbyte/bg3se/blob/main/Docs/API.md#ismodloadedmodguid
  • statFunctions table of [TargetStat|string] = function(partyMemberBeingProcesed, FilterParamMap)
FilterProcessor:RegisterNewFilterProcessor (modUUID, predicateFunction, filterProcessor)
Add a new filter processor -

Parameters:

  • modUUID that ScriptExtender has registered for your mod, for tracking purposes - https://github.com/Norbyte/bg3se/blob/main/Docs/API.md#ismodloadedmodguid will throw an error if the mod identified by that UUID is not loaded
  • predicateFunction function(Filter) Should test the filter to see if the filterProcessor can process it, returning true if so
  • filterProcessor function(CHARACTER, ParamMap) proceses the filter against the provided character, setting ParamMap.winners and optionally ParamMap.winningVal
FilterProcessor:ExecuteFilterAgainstEligiblePartyMembers (filter, prefilters, customItemFilterFields, eligiblePartyMembers, targetsWithAmountWon, inventoryHolder, item, root)
Executes the provided Filter against the provided params. Any exceptions will be logged, swallowed, and whatever the value of the winners table was at exception time will be returned

Parameters:

  • filter the single filter being evaluated, such as a Compare or TargetFilter
  • prefilters associated with the ItemFilter
  • customItemFilterFields any fields that aren't FILTERS or PREFILTERS that were found on the ItemFilter
  • eligiblePartyMembers GUIDSTRING[]
  • targetsWithAmountWon table
  • inventoryHolder CHARACTER
  • item GUIDSTRING
  • root GUIDSTRING

Returns:

    table winners The survivors that were eligible to receive the item, or the original survivors table if none were eligible

Tables

FilterParamMap
The table that's passed to each FilterProcessor

Fields:

  • winners GUIDSTRING[] List of targets that pass the filter - should be set by the FilterProcessor
  • winningVal any value identified by the filter that is currently the victor across all partyMembers
  • targetsWithAmountWon table copy of the winners table across all filters being run for the given item
  • filter the single filter being evaluated, such as a Compare or TargetFilter
  • prefilters associated with the ItemFilter
  • customItemFilterFields any fields that aren't FILTERS or PREFILTERS that were found on the ItemFilter
  • item GUIDSTRING being sorted
  • root GUIDSTRING rootTemplate of the item
  • inventoryHolder CHARACTER
