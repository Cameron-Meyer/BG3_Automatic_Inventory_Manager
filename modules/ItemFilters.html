Functions

RegisterItemFilterMapPreset (modUUID, presetName, itemFilterMaps)
Registers a new Preset with the provided ItemFilterMaps - files will be created, overwriting any existing contents, and the preset information will be added to the PRESETS.FILTERS_PRESETS configuration property.

Parameters:

  • modUUID UUID that ScriptExtender has registered for your mod, for tracking purposes - https://github.com/Norbyte/bg3se/blob/main/Docs/API.md#ismodloadedmodguid will throw an error if the mod identified by that UUID is not loaded
  • presetName string to save the itemFilterMaps under. Your mod name will be automatically prepended to this in {MOD_NAME}-{presetName} format.
  • itemFilterMaps table table of mapName:ItemFilters[] to add

Returns:

    boolean true if the operation succeeded
LoadItemFilterPresets ()
Loads the active ItemFilterMap presets as identified by the PRESETS.ACTIVE_PRESETS configuration property Throws an error if no tables were loaded.

Returns:

    boolean true if at least one requested table was succesfully loaded. Error otherwise.
UpdateItemFilterMapsClone ()
Updates ItemFilters.itemFilterMaps
RegisterItemFilterLookupFunction (modUUID, ...)
Add custom function(s) to use to find ItemFilters for a given item within the available ItemFilterMaps.

Parameters:

  • modUUID that ScriptExtender has registered for your mod, for tracking purposes - https://github.com/Norbyte/bg3se/blob/main/Docs/API.md#ismodloadedmodguid will throw an error if the mod identified by that UUID is not loaded
  • ... function should accept:
    1. table(string, table(string, ItemFilter)) - immutable copy of all the itemFilterMaps to perform lookup against
    2. GUIDSTRING - the root template of the item being sorted
    3. GUIDSTRING - the item being sorted
    4. GUIDSTRING - the inventoryHolder
    and return a list of ItemFilters
SearchForItemFilters (item, root, inventoryHolder)
Finds all ItemFilters for the given item

Parameters:

  • item string
  • root string
  • inventoryHolder string

Returns:

    A consolidated ItemFilter containing all the filters, prefilters, and custom fields found for the given item, with normalized priorities

Tables

ItemFilters.ItemFields.PreFilters
Filters that pre-filter eligible party members before a stack of items, or an item in the stack, is processed.

Fields:

  • STACK_LIMIT Filters out any party members that have > than the specified limit
  • EXCLUDE_PARTY_MEMBERS Array of party members to exclude from processing
  • ENCUMBRANCE Internal only
ItemFilters.FilterFields.CompareStategy
Used by Compare Filters to determine whether a higher or lower value is considered the "winner"

Fields:

  • LOWER
  • HIGHER
ItemFilters.FilterFields.TargetStat
Used by Compare Filters to identify the criteria that party members are filtered on

Fields:

  • HEALTH_PERCENTAGE
  • STACK_AMOUNT the amount of that item's template in the party's inventory
  • PROFICIENCY as dictated by the item, i.e. SleightOfHand for lockpicks
  • WEAPON_SCORE using Osi.WeaponScore, does some math, i have no idea.
  • WEAPON_ABILITY as dictated by the item, i.e. Greatswords use Strength
  • HAS_TYPE_EQUIPPED
  • SKILL_TYPE requires TargetSubStat to be specified
  • ABILITY_STAT requires TargetSubStat to be specified
  • ARMOR_CLASS
